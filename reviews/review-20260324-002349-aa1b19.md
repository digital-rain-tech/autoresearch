## Medium

### 4. `train.py:517`
- File: `train.py:517`
- Severity: medium
- Category: Diff
- Description: The UCB implementation optimizes average raw loss per bucket (`ucb_total_loss / ucb_count`), but ADR-007 specifies average loss reduction / learning progress as the exploitation signal. Rewarding the highest-loss bucket is not the same as rewarding the bucket that teaches the model the most, so the adaptive policy can converge toward persistently hard or noisy buckets instead of the most useful ones.
- Suggested fix: Track per-bucket learning progress, such as reduction relative to each bucket's prior average loss, and use that improvement signal in the UCB score. Add a deterministic unit test that distinguishes "highest loss" from "largest improvement."
